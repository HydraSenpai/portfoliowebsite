JP Morgan Bank API

For class CS217

This project is an API built for users (bank analysts) to search, add and modify customer data. The project also dealt with error, duplicate data and API connection faults. The project was part of class CS217, a J.P Morgan Chase & Co sponsored module. I was able to interact with industry professionals to learn real-world skills and put this API together. The backend is written mostly in java, with other elements implemented to add functionality. The whole project was designed in the Maven framework.

I initally found this project very difficult as being introduced to a brand new framework and new backend concepts is a lot of information to take in and learn in a few weeks. The class was also run very poorly with barely any support, so the whole class was mainly self-learning.

I enjoyed this project as it introduced me to hundreds of new concepts. I learned to work in a scrum team, using agile methodology under strict time limits. I learned many backend concepts such as HTTP calls and DOM display etc, which are all concepts I use in newer projects I create.
